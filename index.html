Некоторые базовые фигуры SVG

<tag> Рисует Можно задавать
<rect> прямоугольник положение, ширину и высоту, радиус углов.
<circle> окружность радиус, положение центра.
<line> прямая линия положения точек 1 и 2
<polyline> ломаная линия координаты каждой точки
<polygon> многоугольник координаты списка точек
MDN web documentation

Итого

  • CSS images полезны для самых простых изображений
  • Для сложных картинок лучше использовать SVG
  • В начале пути достаточно:
    • уметь сделать базовые иконки меню на CSS
    • знать основы SVG: заменить заливку или обводку
    • уметь пользоваться графическими редакторами чтобы экспортировать графику

Анимация векторной графики

Анимация позволяет элементу постепенно переходить от одного состояния к другому.

  1. CSS animations
  2. JS animations
  3. Multimedia Integration Language (SMIL) animations
  4. Web animations API*

CSS animations

  • Необходимо в начале указать ключевые кадры (@keyframes)
  • @keyframes содержат стили, которые будут применены к элементу на каждом из шагов.
  • Вы можете изменить CSS свойства столько раз, сколько захотите.

CSS animations

            <div class="wrap-ball">
              <div class="ball"></div>
            </div>
See demo

CSS animations

animation-name название анимации
animation-duration продолжительность анимации
animation-delay задержка начала анимации
animation-iteration-count количество итераций, которое анимация отрабатывает

CSS animations

animation-direction направление анимации
animation-timing-function скорость анимации: линейная,
в кубической функции Безье (см easings.net)
animation-fill-mode стили элемента вне анимации
animation-play-state позволяет приостановить анимацию

CSS animations

JS animations

  • JavaScript-анимация применяется там, где не подходит CSS.
  • Можно создавать движение по сложной траектории
  • С JS можно добавлять и удалять элементы
  • Создавать сложнее, чем CSS
  • Может перегрузить браузер

Synchronized Multimedia Integration Language (SMIL) animations

  • Вставляется напрямую в SVG
  • <animate> меняет скалярные величины
  • <set> изменяет атрибуты преобразования
  • <animateMotion> позволяет следовать траектории движения
  • <animateColor> меняет цветовые атрибуты

SMIL animations

          <svg width="100" height="300">
            <circle id="circle" ...">
            <animate ...
            attributeName="cy"
            values="250; 50; 250; 150; 250"
            keyTimes="0; 0.2; 0.4; 0.6; 1"
            dur="1.5s" begin="click"/>
          </svg>

Web animations API

  • Позволяет нам вынести CSS animations и CSS transitions в JS
  • Широкий спектр возможных манипуляций
  • Поддержка динамически меняющихся значений
  • Хорошая производительность
  • Курс 'Встречайте Web Animations API'

Web animations API

Итого

  • Многие необходимые в работе анимации можно сделать с помощью CSS
  • JS animations – помимо CSS преобразований можно еще и удалять элементы, создавать новые
  • SMIL чуть более функционален, но постепенно уходит из применения
  • Пора присмотреться к Web Animations API

Немного про Canvas

  • Грубо говоря, это 'холст', на котором можно управлять каждым пикселем
  • Можно вставить любую картинку или нарисовать что-то
  • При масшабировании качество может снизиться
  • Для анимации браузеру приходится перерисовывать все изображение
  • Изображение не добавляеся в DOM и не хранится в памяти браузера
